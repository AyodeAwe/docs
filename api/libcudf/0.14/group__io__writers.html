std::shared_ptr<detail::orc::orc_chunked_state> cudf::io::write_orc_chunked_begin ( write_orc_chunked_args const &  args,
rmm::mr::device_memory_resource *  mr = rmm::mr::get_default_resource() 
)

Begin the process of writing an ORC file in a chunked/stream form.

The intent of the write_orc_chunked_ path is to allow writing of an arbitrarily large / arbitrary number of rows to an ORC file in multiple passes.

The following code snippet demonstrates how to write a single ORC file containing one logical table by writing a series of individual cudf::tables.

...
std::string filepath = "dataset.orc";
cudf::io::write_orc_chunked_args args{cudf::sink_info(filepath), table->view()};
...
auto state = cudf::write_orc_chunked_begin(args);
cudf::write_orc_chunked(table0, state);
cudf::write_orc_chunked(table1, state);
...
cudf_write_orc_chunked_end(state);
Parameters
[in]argsSettings for controlling writing behavior
[in]mrDevice memory resource to use for device memory allocation
Returns
pointer to an anonymous state structure storing information about the chunked write. this pointer must be passed to all subsequent write_orc_chunked() and write_orc_chunked_end() calls.
