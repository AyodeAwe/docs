Detailed Description

Derived class of table_metadata which includes nullability information per column of input.

This information is used as an optimization for chunked writes. If the caller leaves column_nullable uninitialized, the writer code will assume the worst case : that all columns are nullable.

If the column_nullable field is not empty, it is expected that it has a length equal to the number of columns in the table being written.

In the case where column nullability is known, pass true if the corresponding column could contain nulls in one or more subtables to be written, otherwise false.
