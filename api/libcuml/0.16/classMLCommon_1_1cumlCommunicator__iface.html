Detailed Description

cuML communicator plugin interface. This is part of ml-prims because multi GPU ml-prims need it and we want to avoid a dependency of ml-prims to cuML. This is not part of cuML-comms as we would have a circular dependency of cuML-comms and cuML/ml-prims (cuML-comms to cuML/ml-prims due to the implementation of initialize_mpi_comms in cuML-comms, which depdends on cumlHandle and cuML/ml-prims to cuML-comms to cumlCommunicator_iface if cumlCommunicator_iface would be part of cuML-comms).
