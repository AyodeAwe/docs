Assign cache location to a set of keys using LRU replacement policy.

The keys and the corresponding cache_set arrays shall be sorted according to cache_set in ascending order. One block should be launched for every cache set.

Each cache set is sorted according to time_stamp, and values from keys are filled in starting at the oldest time stamp. Enties that were accessed at the current time are not reassigned.

Template Parameters
nthreads number of threads per block
assaciativity number of keys in a cache set
Parameters
[in] keys that we want to cache size [n]
[in] n number of keys
[in] cache_set assigned to keys, size [n]
[in,out] cached_keys keys of already cached vectors, size [n_cache_sets*associativity], on exit it will be updated with the cached elements from keys.
[in] n_cache_sets number of cache sets
[in,out] cache_time will be updated to "time" for those elements that could be assigned to a cache location, size [n_cache_sets*associativity]
[in] time time stamp
[out] cache_idx the cache idx assigned to the input, or -1 if it could not be cached, size [n]

Collect vectors of data from the cache into a contiguous memory buffer.

We assume contiguous memory layout for the output buffer, i.e. we get column vectors into a column major out buffer, or row vectors into a row major output buffer.

On exit, the output array is filled the following way: out[i + n_vec*k] = cache[i + n_vec * cache_idx[k]]), where i=0..n_vec-1, and k = 0..n-1 where cache_idx[k] >= 0

We ignore vectors where cache_idx[k] < 0.

Parameters
[in] cache stores the cached data, size [n_vec x n_cached_vectors]
[in] n_vec number of elements in a cached vector
[in] cache_idx cache indices, size [n]
[in] n the number of elements that need to be collected
[out] out vectors collected from the cache, size [n_vec * n]

int n_cache_sets ,
int * rank

Rank the entries in a cache set according the time stamp, return the indices that would sort the time stamp in ascending order.

Assume we have a single cache set with time stamps as: key (threadIdx.x): 0 1 2 3 val (time stamp): 8 6 7 5

The corresponding sorted key-value pairs: key: 3 1 2 0 val: 5 6 7 8 rank: 0th 1st 2nd 3rd

On return, the rank is assigned for each thread: threadIdx.x: 0 1 2 3 rank: 3 1 2 0

For multiple cache sets, launch one block per cache set.

Template Parameters
nthreads number of threads per block (nthreads <= associativity)
associativity number of items in a cache set
Parameters
[in] cache_time time stamp of caching the data, size [associativity * n_cache_sets]
[in] n_cache_sets number of cache sets
[out] rank within the cache set size [nthreads * items_per_thread] Each block should give a different pointer for rank.

Store vectors of data into the cache.

Elements within a vector should be contiguous in memory (i.e. column vectors for column major data storage, or row vectors of row major data).

If tile_idx==nullptr then the operation is the opposite of get_vecs, i.e. we store cache[i + cache_idx[k]*n_vec] = tile[i + k*n_vec], for i=0..n_vec-1, k=0..n-1

If tile_idx != nullptr, then we permute the vectors from tile according to tile_idx. This allows to store vectors from a buffer where the individual vectors are not stored contiguously (but the elements of each vector shall be contiguous): cache[i + cache_idx[k]*n_vec] = tile[i + tile_idx[k]*n_vec], for i=0..n_vec-1, k=0..n-1

Parameters
[in] tile stores the data to be cashed cached, size [n_vec x n_tile]
[in] n_tile number of vectors in the input tile
[in] n_vec number of elements in a cached vector
[in] tile_idx indices of vectors that we want to store
[in] n number of vectos that we want to store (n <= n_tile)
[in] cache_idx cache indices, size [n], negative values are ignored
[in,out] cache updated cache
[in] n_cache_vecs
