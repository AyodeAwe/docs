Search the kNN for the k-nearest neighbors of a set of query vectors

Parameters
input vector of device device memory array pointers to search
sizes vector of memory sizes for each device array pointer in input
D number of cols in input and search_items
search_items set of vectors to query for neighbors
n number of items in search_items
res_I pointer to device memory for returning k nearest indices
res_D pointer to device memory for returning k nearest distances
k number of neighbors to query
allocator the device memory allocator to use for temporary scratch memory
userStream the main cuda stream to use
internalStreams optional when n_params > 0, the index partitions can be queried in parallel using these streams. Note that n_int_streams also has to be > 0 for these to be used and their cardinality does not need to correspond to n_parts.
n_int_streams size of internalStreams. When this is <= 0, only the user stream will be used.
rowMajorIndex are the index arrays in row-major layout?
rowMajorQuery are the query array in row-major layout?
translations translation ids for indices when index rows represent non-contiguous partitions

KNN classifier using voting based on the statistical mode of classes. In the event of a tie, the class with the lowest index in the sorted array of unique monotonically increasing labels will be used.

Template Parameters
TPB_X the number of threads per block to use
Parameters
out output array of size (n_samples * y.size())
knn_indices index array from knn search
y vector of label arrays. for multilabel classification, each element in the vector is a different "output" array of labels corresponding to the i'th output.
n_labels number of vertices in index (eg. size of each y array)
n_rows number of rows in knn_indices
k number of neighbors in knn_indices
uniq_labels vector of the sorted unique labels for each array in y
n_unique vector of sizes for each array in uniq_labels
allocator device allocator to use for temporary workspace
user_stream main stream to use for queuing isolated CUDA events
int_streams internal streams to use for parallelizing independent CUDA events.
n_int_streams number of elements in int_streams array. If this is less than 1, the user_stream is used.

Compute class probabilities

Note: Since class_probs will use the same round robin strategy for distributing work to the streams, we don't need to explicitly synchronize the streams here.

Choose max probability
