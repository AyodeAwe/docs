template<typename MatrixIteratorT, typename MapIteratorT>
void gather(const MatrixIteratorT in, int D, int N, MapIteratorT map, int map_length, MatrixIteratorT out, cudaStream_t stream)¶

gather copies rows from a source matrix into a destination matrix according to a map.

Template Parameters

  • MatrixIteratorT – Random-access iterator type, for reading input matrix (may be a simple pointer type).

  • MapIteratorT – Random-access iterator type, for reading input map (may be a simple pointer type).

Parameters

  • in – Pointer to the input matrix (assumed to be row-major)

  • D – Leading dimension of the input matrix ‘in’, which in-case of row-major storage is the number of columns

  • N – Second dimension

  • map – Pointer to the input sequence of gather locations

  • map_length – The length of ‘map’ and ‘stencil’

  • out – Pointer to the output matrix (assumed to be row-major)

  • stream – CUDA stream to launch kernels within

template<typename m_t, typename idx_t = int, typename Lambda, typename ...Vecs>
void linewiseOp(m_t *out, const m_t *in, const idx_t lineLen, const idx_t nLines, const bool alongLines, Lambda op, cudaStream_t stream, Vecs... vecs)¶

Run a function over matrix lines (rows or columns) with a variable number row-vectors or column-vectors. The term line here signifies that the lines can be either columns or rows, depending on the matrix layout. What matters is if the vectors are applied along lines (indices of vectors correspond to indices within lines), or across lines (indices of vectors correspond to line numbers).

Parameters

  • out – [out] result of the operation; can be same as in; should be aligned the same as in to allow faster vectorized memory transfers.

  • in – [in] input matrix consisting of nLines lines, each lineLen-long.

  • lineLen – [in] length of matrix line in elements (=nCols in row-major or =nRows in col-major)

  • nLines – [in] number of matrix lines (=nRows in row-major or =nCols in col-major)

  • alongLines – [in] whether vectors are indices along or across lines.

  • op – [in] the operation applied on each line: for i in [0..lineLen) and j in [0..nLines): out[i, j] = op(in[i, j], vec1[i], vec2[i], … veck[i]) if alongLines = true out[i, j] = op(in[i, j], vec1[j], vec2[j], … veck[j]) if alongLines = false where matrix indexing is row-major ([i, j] = [i + lineLen * j]).

  • stream – [in] a cuda stream for the kernels

  • vecs – [in] zero or more vectors to be passed as arguments, size of each vector is alongLines ? lineLen : nLines.
